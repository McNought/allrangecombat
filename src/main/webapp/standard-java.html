A Downloadable Development Environment for IPL

An easy way to set up developement environments for projects.
Easy and modern (sets up tools for a VM).

Using the installer script

It is possible to install the entire developement environment software and set up the environment from a single command.

Simply paste the following into a command prompt:

@powershell -NoProfile -ExecutionPolicy Bypass -Command "((new-object net.webclient).DownloadString('http:///scripts/server/install.ps1')) | iex"

Or from within powershell:

((new-object net.webclient).DownloadString('http:///scripts/server/install.ps1')) | iex

Options

Options for configuring the install script are stored in environment variables.

  • DDSDomain
    Override the DDS domain derived from the last 2 digits of the machines IP address. Useful for developer machines where ip addresses should come from [the wiki](http://mirage/wiki/doku.php?id=i1284007:unique_inspire.dds.domain_values) or when using two machines together.
  • IMPADDomain
    Override the default IMP Active Directory Domain used to authenticate users, default INSPIRE.local.

Scripts

When installed you will end up with all the software installed into `C:\INSPIRE`.

In this folder you will find the following scripts to start or stop various elements of the system:

  1. start-simulator.bat
    starts the simulator.

Folders

When installed all software will be in `C:\Inspire`.

The structure will be as follows:

  1. C:\Inspire
    All scripts
  2. C:\Inspire\7zip
    All scripts
  3. C:\Inspire\java
    The current installed version of JAVA
  4. C:\Inspire\mysql
    All scripts
  5. C:\Inspire\flyway
    Flyway database creation utility used during installation.
  6. C:\Inspire\tomcat
    The currently installed version of Tomcat. Not initially running (will have IMP in WEBAPPS).
